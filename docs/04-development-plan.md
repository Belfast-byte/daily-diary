# 分步开发计划

## 阶段 0：工程初始化

目标：创建可运行的 Android 工程骨架。

任务：

- 创建 Kotlin Android 项目。
- 配置 Gradle、Compose、Material 3。
- 配置 Hilt、Room、DataStore、WorkManager、Biometric。
- 建立包结构。
- 建立基础主题和底部导航。

验收：

- App 能在模拟器或真机启动。
- 底部导航能切换空页面。
- 单元测试任务能运行。

## 阶段 1：数据层

目标：完成本地加密数据库和核心 Repository。

任务：

- 实现 `DiaryEntry`、`ActivityTag`、`DiaryEntryTagCrossRef`、`Attachment` 实体。
- 实现 DAO 查询。
- 接入 SQLCipher。
- 实现数据库密钥管理。
- 实现 `DiaryRepository`。
- 添加 Room 迁移测试。

验收：

- 可以保存、读取、更新当天日记。
- 每天只能有一条主日记。
- 数据库重启后数据仍存在。
- 数据库打开失败能暴露错误。

## 阶段 2：今日记录

目标：用户可以完成核心记录闭环。

任务：

- 实现 `TodayScreen`。
- 实现心情选择组件。
- 实现正文输入。
- 实现活动标签多选。
- 实现保存和编辑当天记录。
- 保存失败时保留输入。

验收：

- 新用户能选择心情并保存。
- 当天再次打开显示已有记录。
- 修改正文后能更新同一条记录。
- 空正文但有心情可以保存。

## 阶段 3：日历与历史

目标：用户可以回看和检索记录。

任务：

- 实现月历视图。
- 月历日期展示记录状态和心情颜色。
- 实现日期详情入口。
- 实现历史时间线。
- 实现关键词搜索。
- 实现心情筛选。

验收：

- 月视图能正确展示当前月记录。
- 点击日期能进入对应记录。
- 搜索能返回包含关键词的记录。
- 筛选结果与所选心情一致。

## 阶段 4：统计

目标：用户可以看到基础情绪趋势。

任务：

- 实现近 7 天心情分布。
- 实现近 30 天趋势。
- 实现总记录天数。
- 实现连续记录天数。
- 添加统计 UseCase 单元测试。

验收：

- 空数据时显示空状态。
- 有数据时统计结果准确。
- 日期跨月时连续天数计算正确。

## 阶段 5：隐私锁与设置

目标：完成隐私保护和个性化设置。

任务：

- 实现启动解锁流程。
- 接入 AndroidX Biometric。
- 实现隐私锁开关。
- 实现提醒时间设置。
- 使用 DataStore 保存设置。

验收：

- 默认需要解锁才能进入主界面。
- 解锁失败不能进入主界面。
- 关闭隐私锁前需要先解锁。
- 设置重启后仍保留。

## 阶段 6：提醒与导出

目标：补齐日常使用能力和数据迁移能力。

任务：

- 申请通知权限。
- 使用 WorkManager 调度每日提醒。
- 实现 JSON 导出。
- 实现 CSV 导出。
- 导出失败显示明确错误。

验收：

- 用户开启提醒后能调度每日任务。
- 关闭提醒后任务取消。
- JSON 导出能被解析。
- CSV 导出字段完整。

## 阶段 7：发布前验证

目标：确保 MVP 可以稳定交付。

任务：

- 补齐关键单元测试。
- 补齐 Compose UI 测试。
- 进行数据库迁移验证。
- 检查权限和隐私说明。
- 准备发布配置。

验收：

- 自动化测试通过。
- App 在冷启动、重启、断网情况下可用。
- 隐私锁、保存、搜索、导出主路径可用。
- 无敏感内容日志输出。
