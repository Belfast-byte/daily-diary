# 安全与隐私

## 数据边界

MVP 不上传任何日记数据。全部日记正文、心情、标签和附件元数据只保存在设备本地。

## 加密策略

- Room 数据库使用 SQLCipher 加密。
- 数据库密钥随机生成。
- 数据库密钥由 Android Keystore 保护。
- 如果密钥读取失败，App 不创建新空库替代旧库。
- 数据库打开失败时显示明确错误并阻止进入主界面。

## 解锁策略

默认启用隐私锁。

- 支持 AndroidX Biometric。
- 允许生物识别和设备凭据。
- 解锁成功后进入主界面。
- 解锁失败后保持锁定。
- 用户关闭隐私锁时必须先完成一次解锁。

## 导出策略

导出是显式用户操作。

- JSON 导出包含完整结构化数据。
- CSV 导出包含日期、心情、正文和标签。
- 导出文件不默认加密。
- 导出前显示文件包含明文日记内容。
- 导出失败必须显示失败原因。

## 权限策略

MVP 尽量不申请危险权限。

- 不读取通讯录。
- 不读取定位。
- 不读取相册。
- 不录音。
- 不访问网络。

每日提醒使用系统通知权限。Android 13 及以上需要用户授权通知权限。

## 日志策略

禁止在日志中输出：

- 日记正文。
- 数据库密钥。
- 导出文件内容。
- 解锁失败的系统细节。

允许输出：

- 页面生命周期事件。
- 非敏感错误类型。
- 数据库版本号。
- 导出任务是否完成。

## 隐私说明

App 内需要提供简短隐私说明：

- 数据默认保存在本机。
- App 不上传日记内容。
- 用户可以导出数据。
- 导出文件由用户自行保管。
- 卸载 App 可能删除本地数据。
